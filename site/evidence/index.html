Polinko publishes tracked results with their task shape, exclusions, limitations, and source notes. Each score points to the cases and records needed to understand what it supports.

How to read the evidence

Each result applies to a named evaluation surface.

  • Counts apply to the named tracked cases and snapshot.
  • A pass establishes only the bounded case result.
  • Retained failures, evictions, exclusions, and first-attempt instability remain part of the result.
  • Raw local databases, exports, and private transcripts require explicit selection before promotion.
  • Public notes point back to cases, reports, code, or manifests where available.

What the cases represent

The cases test observable departures from evidence, constraints, and uncertainty.

Source departure

The response invents, misattributes, or claims support absent from the available source material.

Constraint departure

The response ignores an explicit request for directness, exact output, sparse control, or a mid-thread mode change.

Uncertainty departure

The response claims knowledge, memory, verification, or completed action beyond the system's evidence.

Operator burden

The response adds commentary, interpretation, or advice that creates correction work instead of advancing the stated task.

Failure evidence

Failures are either retained as evidence or removed from the case set.

Retained failure

The case is valid and the mismatch is in scope. It remains evidence because it reveals a real seam in model behaviour, case design, or method.

Evicted failure

The case is malformed, duplicated, irrelevant, or dominated by upstream noise. Removing it protects the evidence set from false pressure.

Eviction is an explicit upstream quality decision with a reviewable reason.

Public evidence boundary

The public record is curated from a larger working environment.

Tracked public evidence includes stable reports, selected case files, manifests, diagrams, and dated research notes. Raw databases, local exports, private transcripts, scratch screenshots, and uncurated evidence dumps remain local unless they are explicitly reviewed and promoted.

This protects privacy and keeps the public record focused enough to review. The website is a reading layer rather than the complete research environment.
