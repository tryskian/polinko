Research portfolio

Polinko currently tracks six behavioural research lanes.

The lanes differ in maturity and purpose. Some have stable tracked evals, some are promoted or operationalised, and one remains deliberately thin. Their status records the evidence each lane has earned.

How to read status

Status describes maturity, not importance.

Stable baseline
Repeatable on the current set, with the next pressure boundary named explicitly.
Promoted
A tracked lane has enough repeated signal to support a current research conclusion.
Operationalised
A bounded behaviour is tested through repeatable cases and reports.
Thin
Distinct evidence exists, but the lane is still too narrow for a stronger claim.
Staged
A method contract is defined before new evidence is cut.
Historical
The idea remains in the record but is no longer part of the current method.

Stable baseline

OCR generalisation

Can OCR reliability transfer from stabilised replay to new visual conditions while preserving the first-gate contract?

Current read

The current image set is stable: 25/25 growth cases, 16/16 focus cases, and no active fail-history cases. This is the baseline for broader generalisation tests.

Evidence boundary

Current-set replay does not establish broad OCR robustness. New layouts, crops, contrast levels, density, and ambiguous visual cues remain untested.

Next test

Curate a broader case set while keeping the OCR runtime and first pass / fail gate fixed. Failed cases are then retained as in-scope evidence or evicted as malformed, duplicate, or noisy source material.

Source: Beta 2.3 snapshot

Promoted lane

Co-reasoning reliability

Can collaboration behaviours become inspectable binary eval signals instead of remaining informal style judgements?

Current read

Co-reasoning is the first promoted non-OCR lane. Its tracked surface covers flexible constraint retention, mid-thread mode shifts, bounded style adaptation, grounded playful abstraction, and collaboration that does more than summarise.

Evidence

The current tracked style surface closed at 14/14 with high confidence across all cases.

Boundary

The result applies to the tracked stress cases. Broader collaboration quality, creativity, and user satisfaction remain outside this lane.

Next move

Keep weaker export cues below claim status until repeated evidence earns promotion.

Source: Co-reasoning promotion snapshot

Operationalised

Retrieval grounding

Can retrieval stay inspectable and source-bound across global recall, session isolation, and scoped file search?

Current read

Retrieval is operationalised across two connected surfaces: global retrieval and file search. The tracked cases include OCR-seeded recall, session isolation, PDF-grounded lookup, scoped search, and image-context smoke.

Evidence

The tracked snapshot reports 12/12 retrieval passes and 5/5 file-search passes, with zero recorded miss or leak in that run.

Boundary

This is a bounded functional result. Privacy and security assurance require separate tests.

Next move

Keep the lane stable until a distinct miss, leak, or source-grounding pressure earns another investigation.

Source: Retrieval grounding snapshot

Operationalised

Response-behaviour stability

Can directness, uncertainty, and bounded claims remain visible as deterministic evaluation behaviour?

Current read

The lane measures concrete behaviours: absence of false action claims after repo requests, explicit uncertainty when context is missing, avoidance of invented live verification, refusal of pretended memory, and bounded role behaviour.

Evidence

The tracked snapshot closed at 7/7. One case failed on its first attempt before recovering on the next two attempts, and that wobble remains visible.

Boundary

The cases demonstrate bounded response behaviour under a deterministic surface. Broader personality and safety profiles remain outside scope.

Next move

Widen only when a distinct response failure family appears. Keep recovered instability visible inside the final pass count.

Source: Response-behaviour stability snapshot

Operationalised

Uncertainty and hallucination boundaries

Can explicit uncertainty prevent unsupported invention without blocking a grounded discussion of the claim being tested?

Current read

The tracked lane covers invented events, relationship-motive guesses, archive-lore authorship claims, and fabricated archive-discipline records.

Evidence

The tracked surface closed at 9/9 across nine cases, including two export-backed invention families.

Boundary

The result keeps hallucination as an active boundary while showing that one binary gate holds across a defined set of unsupported claim shapes.

Next move

Promote only genuinely distinct invention families. Duplicate variants should stay grouped rather than broaden coverage.

Source: Hallucination-boundary promotion

Thin lane

Operator burden

When does commentary, interpretation, or advisory drift make direct work harder for the person operating the system?

Current read

Low-burden execution can hold under sparse-control, raw-pull, protocol-lock, and exact-output boundaries. Real failure pressure appears when the system drifts into interpretation or advice outside the task.

Evidence

The current row-local surface contains four pass anchors, two retained fail rows, and one evicted fail row.

Boundary

The lane is narrow. Its current backlog is dominated by duplicate families, so more rows would create false breadth rather than stronger evidence.

Next move

Wait for a distinct task shape or improve the visibility of the current cases. Keep duplicate examples out of fresh evidence.

Source: Operator burden signal shape

Forward method

New work follows the source-first evidence chain.

Polinko preserves row or case evidence before producing lane summaries or public claims. Earlier method notes remain in the source record, but the active work moves through a human-led collaborative process with explicit evidence boundaries, staged tests, and human claim review.

The next public work should make source artefacts, row or case judgements, and lane summaries inspectable before any new result is promoted.
