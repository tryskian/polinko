# Polinko

[![CI](https://github.com/tryskian/polinko/actions/workflows/ci.yml/badge.svg)](https://github.com/tryskian/polinko/actions/workflows/ci.yml)
![Binary Eval](https://img.shields.io/badge/eval-binary_pass%2Ffail-4E79A7)
![Research Surface](https://img.shields.io/badge/research-repo_native-76B7B2)

Polinko exists to make AI failure inspectable enough to act on. It is a
human-led, AI-assisted, repo-native evaluation lab with a local-first FastAPI
backend, OCR-forward reliability loops, binary eval gates, and evidence-first
documentation.

The website is a lightweight doorway into the work. The repository is the
research surface.

## What This Repo Is

- Backend-first runtime:
  - FastAPI API + CLI are canonical.
- Research object:
  - tracked docs, tests, evals, and diagrams are the main research surface.
- Public doorway:
  - `https://www.krystian.io/` points into the work; it does not recreate the
    research system.
- Collaboration model:
  - research direction, evidence interpretation, and publication decisions are
    human-led
  - OpenAI Codex is used for repo-local engineering collaboration

## Quick Start

From repo root:

```bash
make deps-install
cp .env.example .env
# set OPENAI_API_KEY in .env
make doctor-env
make server
```

Open:

- `http://127.0.0.1:8000/docs`

Portfolio doorway:

```bash
make portfolio
```

`make portfolio` rebuilds/serves and prints a cache-busted URL. It does not
open a browser by default.

## Public Surface

- `GET /` redirects to `GET /portfolio`
- `GET /portfolio` serves the public doorway
- static Netlify build is generated by:
  - `python3 tools/build_portfolio_static.py`

Current production URL:

- [www.krystian.io](https://www.krystian.io/)

## Repo Map

- `api/` API implementation
- `core/` runtime logic
- `tools/` eval and maintenance scripts
- `tests/` test suite
- `docs/public/` curated public reading path
- `docs/governance/` charter, state, decisions
- `docs/runtime/` runbook and architecture
- `docs/eval/` eval evidence and phase context

## Notes

- Private/local research materials stay untracked by design.
- `frontend/` and `ui/` are local-only scaffolds unless deliberately restored.
- Static D3/SVG evidence diagrams should live beside Mermaid diagrams in the
  repo research lane, not as portfolio UI.
