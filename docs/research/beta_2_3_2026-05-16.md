# Beta 2.3 Snapshot

Date: `2026-05-16`

## Boundary

`Beta 2.3` is Polinko's current method beta.

This is the point where OCR moves from stabilized same-image replay into
generalization pressure with explicit OCR-local case governance.

## What Makes This A Beta Boundary

- OCR is no longer proving only repeatability on the current image set
- transcript mining now feeds fresher OCR cases
- OCR-ready generalization candidates now widen intake beyond
  correction-shaped transcript episodes
- OCR follow-up now uses `retain` / `evict` inside the lane instead of simple
  watchlist parking
- the lane can now separate real OCR seams from malformed or duplicate case
  noise

## Method

- start from the stabilized OCR base:
  - growth stability: `25/25`
  - focus stability: `16/16`
  - fail-history cohort: `0` active cases
- mine fresh OCR-framed transcript data
- surface OCR-ready generalization candidates that did not enter
  transcript-mined OCR episodes
- curate a broader case set from new images and more complex visual
  conditions:
  - layout
  - crop quality
  - contrast
  - density
  - cue ambiguity
- keep the OCR runtime and first-gate eval contract fixed
- judge the first gate as `pass` / `fail`
- judge failed rows as `retain` / `evict`
- rerun only after retained seams and corrected cases are explicit

## Current Read

- the stabilized OCR base remains real and useful
- the next OCR kernel is broader OCR generalization pressure
- `retain` keeps true OCR seams as live evidence
- `evict` removes malformed or duplicate case noise upstream

## Why It Matters

Polinko now tests OCR maturity under changed visual conditions instead of
treating current-set replay stability as the end of the claim.
