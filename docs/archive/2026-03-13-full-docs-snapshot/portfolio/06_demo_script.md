# Demo Script (Hybrid, 5-7 Minutes)

Goal: demonstrate one coherent loop:
observation -> theory -> method -> build trace -> evidence.

## Opening (30-45s)

Script:
"This portfolio treats AI behavior as an engineering-research problem.  
I start with interaction observations, translate them into mechanism hypotheses, implement controls, and validate through eval evidence."

Optional framing line:
"It is like trading a mirror for its reflection because the mirror smiled back."

Use framing once, then move to evidence.

## Segment 1 (Section 1): Theory Lens to Mechanism (90s)

Script:
"Section 1 asks why users perceive relationship and agency in coherent model behavior.  
The claim is not that the model has interiority. The claim is that specific system surfaces produce stable cues users interpret socially."

Show:
- one transcript excerpt with coded notes
- the mapped mechanism row(s) in `03_evidence_log.md` (`S1-01`, `S1-02`)

Close line:
"Interpretation is real at interaction level, and still explainable at system level."

## Segment 2 (Section 2): Boundary Protocols (90s)

Script:
"Section 2 tests whether explicit role boundaries and handoff rules improve collaboration quality.  
When boundary logic is explicit, drift is diagnosable and recoverable instead of compounding."

Show:
- before/after boundary case
- session hygiene surfaces (`/deprecate`, `/session/reset`)
- evidence IDs (`S2-01`, `S2-02`)

Close line:
"The main variable is protocol quality, not conversational charm."

## Segment 3 (Section 3): Build + Eval Proof (120s)

Script:
"Section 3 is the implementation lab: each behavioral claim is accepted only if a concrete surface and test artifact exist."

Show:
- `make eval-style`
- `make eval-retrieval`
- `make eval-file-search`
- `make eval-hallucination`
- `make eval-ocr`
- `make quality-gate`

Point to evidence IDs:
- `S3-01` to `S3-04`

Close line:
"This moves the work from anecdote to reproducible behavioral engineering."

## Closing (30s)

Script:
"The contribution is a repeatable method: theory that survives implementation and evidence review.  
Same artifacts, different lenses: research engineer or applied behavioral research."

## Audience Toggle (Use Same Demo Assets)

- Research Engineer audience:
  - emphasize harness design, gates, and runtime/eval separation.
- Applied Behavioral Research audience:
  - emphasize interpretation boundaries, protocol effects, and claim discipline.

## Q&A Bank

- "What counts as evidence vs framing in your portfolio?"
- "How do you avoid over-fitting behavior to one transcript style?"
- "Where do you separate runtime constraints from eval-only constraints?"
- "What are the limits of your current evidence base?"
